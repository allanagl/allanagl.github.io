    <!-- Application Structure Plan: The application is designed as a single-page interactive timeline. This structure was chosen to transform the static, tabular source report into a dynamic and user-centric experience. A vertical timeline is inherently more responsive and scannable than a wide table. The key interaction is a role-based filter, which allows users to instantly see their specific responsibilities within the larger process. This task-oriented design directly supports the goal of making the workflow easily consumable for every team member involved. -->

    <!-- Visualization & Content Choices: 

        - Workflow Steps: Report Info -> 8 procedural steps in a table -> Goal: Organize/Inform -> Viz/Presentation Method: Vertical timeline with expandable cards -> Interaction: Clicking a step header reveals detailed information (Input, Action, Output). Clicking a role filter highlights relevant steps. -> Justification: This layered approach keeps the primary view clean while providing deep detail on demand. Filtering makes the complex process personally relevant to the user. -> Library/Method: HTML/CSS with Tailwind + Vanilla JS.

        - Roles & Details: Report Info -> Columns for Role, Time, Input, etc. -> Goal: Inform -> Viz/Presentation Method: Styled text blocks with icons within each expandable card -> Interaction: Revealed on click. -> Justification: Using icons and clear typography for these details makes them highly scannable and easy to digest once a user drills down into a specific step. -> Library/Method: HTML/CSS with Tailwind + Unicode icons. -->
